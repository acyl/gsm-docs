EXTENDED GSM COMMANDS

This chapter describes the GSM commands supported by the GSM/PSTN cards and their operation. All commands can be written in upper or lower case or a combination of both.

NOTE: The supported and selected options of each GSM command vary depending on the type of handset used. You can ask the modem to indicate the parameters which are:
- selected for each GSM command with <command>?
- supported by each GSM command with <command> =?

Call control commands

Selecting GSM communication parameters +CBST
(supported by all GSM versions)


Parameter command syntax
Command Possible response(s)
+CBST=[<speed>[,<name>[,<ce>]]]  
+CBST? +CBST: <speed>[,<name>,<ce>]
+CBST=? +CBST: (list of supported <speed>s),(list of supported <name>s),(list of supported <ce>s)

Description
This extended command uses 3 parameters to select : the speed, communication mode and transparent mode or RLP (Radio Link Protocol) error correction. Its syntax is as follows:

+CBST=[<speed>[,<name>[,<ce>]]]

NOTE: Parameters that are not filled in (enter simply a comma or a <CR>) retain their previous value.

Selected parameters
Use the following command to find out the values of the selected parameters:

+CBST?
Response: +CBST: <speed>[,<name>,<ce>]
Example: +CBST:7,0,1 (speed : 9600 bps, communication type : asynchronous, RLP error correction activated).

Supported parameters
Use the following command to find out the values of the supported parameters:

+CBST=?
Response: +CBST: (list of supported <speed>s),(list of supported <name>s),(list of supported <ce>s)

Example: +CBST:(0,1,2,3,4,5,6,7,9,65,66,68,70,71),(0),(0,1)

  1. <speed> Decimal number which specifies the chosen speed for a connection in call or answer mode. The possible values are:

0 Automode Automatic speed selection The modem changes the modulation depending on environmental restrictions and range capacities (default value).
1 V.21 300  
2 V.22 1200  
3 V.23 1200/75  
4 V.22bis 2400  
5 V.26ter 2400  
6 V.32 4800  
7 V.32 9600  
8 - No speed specification Relates to specific networks
9 V.32bis 7800  
10 V.32bis 12000  
11 V.32bis 14400  
12 V.34 9600  
13 V.34 12000  
14 V.34 14400  
15 V.34 19200  
16 V.34 28800  
65 V.110 300  
66 V.110 1200  
68 V.110 2400  
70 V.110 4800  
71 V.110 9600  
74 V.110 12000  
75 V.110 14400  
79 V.110 19800  
81 V.110 38400  

For example, if you wish to establish a connection at 2400 bps, <speed> will take the value of 4.
  1. <name> Numerical value which defines the communication mode to be used. The available options are:
<name> Selected option Note(s)
0 Asynchronous Default value
1 Synchronous  
2 PAD Access (asynchronous)  
3 PAD Access (synchronous)  

The default value is 0, which enables asynchronous communication mode.

However, for a connection in synchronous mode, <name> will be set to 1.
  1. <ce> Decimal value which selects RLP (Radio Link Protocol) error correction or Transparent mode. The authorised values are:
<ce> Selected option Note(s)
0 Transparent  
1 RLP Default value

RLP mode is selected by default. If you want to activate transparent mode, the 1 must be replaced by 0 (<ce>=0).

Configuration example for V.32 modulation, asynchronous mode and RLP mode: AT+CBST=7,0,1


Interaction between Modem and GSM AT Commands

AT\Nx also enforces the <ce> element of the +CBST command. AT\Nx updates automatically the +CBST settings but +CBST does not update the \Nx settings !

\Nn Selecting the GSM error correction
 
\N0 Transparent Mode
Connection without error correction handled by the GSM network
\N2 RLP Mode
The modem waits for a RLP connection.
\N3 RLP or transparent Mode
The RLP or transparent modes are authorised.

Phone activity status +CPAS
(only supported by 8110/3110/2110 NOKIA versions)

+CME ERROR: <err>
+CPAS=? +CPAS: (list of supported <pas>s)
+CME ERROR: <err>

Description
Execution command returns the activity status <pas> of the ME. It can be used to interrogate the ME before requesting action from the phone.

Test command returns values supported by the ME as a compound value.

Defined values (Following the GSM card version, the supported values change)

<pas> Selected option
0 Ready (ME allows commands from TA/TE).
1 Unavailable (ME does not allow commands from TA/TE).
2 Unknown (ME is not guaranteed to respond to instructions).
3 Ringing (ME is ready for commands from TA/TE, but the ringer is active).
4 Call in progress (ME is ready for commands from TA/TE, but a call is in progress).
5 Asleep (ME is unable to process commands from TA/TE because it is in a low functionality state).

Read phonebook entries +CPBR
(supported by CMDZ1 SONY, 8110/3110/2110 NOKIA versions)


Action command syntax
Command Possible response(s)
+CPBR=<index1>[,<index2>] +CPBR: <index1>, <number>, <type>,<text>[[...]
<CR><LF>+CPBR: <index2>,<number>,<type>,<text>] +CME ERROR: <err>
+CPBR=? +CPBR: (list of supported <index>s),<nlength>,<tlength>
+CME ERROR: <err>

Description
Execution command returns phonebook entries in location number range <index1>... <index2> from the current phonebook memory storage selected with +CPBS. If <index2> is left out, only location <index1> is returned. Entry fields returned are location number <indexn>, phone number stored there <number> (of format <type>) and text <text> associated with the number. If listing fails in an ME error,
+CME ERROR: <err> is returned.
Test command returns location range supported by the current storage as a compound value and the maximum lengths of <number> and <text> fields. If ME is not currently reachable,
+CME ERROR: <err> is returned.

Defined values
Option Note
<index1>, <index2>, <index> Integer type values in the range of location numbers of phonebook memory from 1 to n
<number> String type phone number of format <type>
<type> Type of address octet in integer format; 145 (international) otherwise 129
<text> String type field of maximum length <tlength>
<nlength> Integer type value indicating the maximum length of field <number>
<tlength> Integer type value indicating the maximum length of field <text>


Write phonebook entry +CPBW
(supported by CMDZ1 SONY, 8110/3110/2110 NOKIA versions)


Action command syntax
Command Possible response(s)
+CPBW=[<index>][,<number> [,<type>[,<text>]]] +CME ERROR: <err>
+CPBW=? +CPBW:(list of supported <index>s),<nlength>, (list of supported <type>s),<tlength>
+CME ERROR: <err>

Description
Execution command writes a phonebook entry in location number <index> in the current phonebook memory storage selected with +CPBS. Entry fields written are phone number <number> (in the format <type>) and text <text> associated with the number. If those fields are omitted, phonebook entry is deleted. If writing fails in an ME error, +CME ERROR: <err> is returned.
Test command returns location range supported by the current storage as a compound value, the maximum length of <number> field, supported number formats of the storage, and the maximum length of <text> field. If ME is not currently reachable, +CME ERROR: <err> is returned. If storage does not offer format information, the format list should be empty parenthesis.

Defined values
Option Note
<index> Integer type values in the range of location numbers of phonebook memory
<number> String type phone number of format <type>
<type> Type of address octet in integer format; default 145 when dialling string includes international access code character '+', otherwise 129
<text> String type field of maximum length <tlength> (1)
<nlength> Integer type value indicating the maximum length of field <number>
<tlength> Integer type value indicating the maximum length of field <text>

(1): The 2110 mobile does not manage the <text> field, if you try to write a text on the SIM card, the text will be replaced by a blank.

Message Receiving and Reading Commands (SMS)

New Message Indications to TE +CNMI
(supported by CMDZ1 SONY, 8110/3110/2110 NOKIA versions)


Parameter command syntax
Command Possible response(s)
+CNMI=[<mode>[,<mt>[,<bm>[,<ds>[,<bfr>]]]]] +CMS ERROR: <err>
+CNMI? +CNMI: <mode>,<mt>,<bm>,<ds>,<bfr>
Default value: 1,0,0,0,0
+CNMI=? +CNMI: (list of supported <mode>s),(list of supported <mt>s),(list of supported <bm>s),(list of supported <ds>s),(list of supported <bfr>s)

Description
Set command selects the procedure, how receiving of new messages from the network is indicated to the TE when TE is active, e.g. DTR signal is ON. <mode> controls the processing of unsolicited result codes specified within this command, <mt> sets the result code indication routing for SMS-DELIVERs, <bm> for CBMs and <ds> for SMS-STATUS-REPORTs. <bfr> defines the handling method for buffered result codes when <mode> 1 is enabled. If ME does not support requested item (although TA does), final result code +CMS ERROR: <err> is returned. The 2110 card can not receive SMS from the mobile.

Test command gives the settings supported by the TA as compound values.

Defined Values for CMDZ1 SONY and 8110/3110 NOKIA versions (For all parameters undescribed, please refer to the GSM Glossary)
<mode> Selected option

<mt> Selected option
0 No SMS-DELIVER indications are routed to the TE
1 Indication of SMS-DELIVER is routed to the TE using unsolicited result code +CMTI: <mem1>,<index> (only for 8110/3110 NOKIA versions)
2 SMS-DELIVERs
+CMT: [<alpha>],<length><CR><LF><pdu> (PDU mode enabled following the GSM card version) or
+CMT: <oa>,[<alpha>],<scts>[,<tooa>,<fo>,<pid>,<dcs>, <sca>,<tosca>,<length>]<CR><LF><data> (about parameters in italics, refer command Show Text Mode Parameters +CSDH)

<bm> Selected option
0 No CBM indications are routed to the TE
2 New CBMs are routed directly to the TE using unsolicited result code
+CBM: <length><CR><LF><pdu> (PDU mode enabled following the GSM card version) or
+CBM: <sn>,<mid>,<dcs>,<page>,<pages><CR><LF><data> (text mode enabled)

<bm> parameter
<bm> Receiving procedure for different message classes
0 Store message to "BM"

<ds> Selected option
0 No SMS-STATUS-REPORTs are routed to the TE

<bfr> Selected option
0 TA buffer of unsolicited result codes defined within this command is flushed to the TE when <mode> 1 is entered (OK response will be given before flushing the codes)

Description (For all parameters undescribed, please refer to the GSM Glossary)
Execution command sends message from a TE to the network (SMS-SUBMIT). Message reference value <mr> is returned to the TE on successful message delivery. Value can be used to identify message upon unsolicited delivery status report result code. If sending fails in a network or an ME error, final result code +CMS ERROR: <err> is returned. This command should be abortable.

-   entered is sent to address <da>
-   text can be typed after the character ">" has been displayed.
-   sending can be cancelled by giving <ESC> character (IRA 27)
-   <ctrl-Z> (IRA 26) must be used to indicate the ending of the message body

+CMS ERROR: <err>
+CMGS=? OK

Description (For all parameters undescribed, please refer to the GSM Glossary)
Execution command sends message from a TE to the network (SMS-SUBMIT). Message reference value <mr> is returned to the TE on successful message delivery. If sending fails in a network or an ME error, final result code +CMS ERROR: <err> is returned. This command should be abortable.

-   <length> must indicate the number of octets coded in the TP layer data unit to be given (i.e. SMSC address octets are excluded)
-   When the card will be ready to accept the PDU, the ">" will be displayed
-   The PDU will be hexadecimal format (similarly as specified for <pdu>) and given in one line; ME/TA converts this coding into the actual octets of PDU
-   When the length octet of the SMSC address (given in the <pdu>) equals zero, the SMSC address set with command Service Centre Address +CSCA is used
-   Sending can be cancelled by giving <ESC> character (IRA 27)
-   <ctrl-Z> (IRA 26) must be used to indicate the ending of PDU
